# fastapi-plugin

![Version](https://img.shields.io/badge/version-0.1.0-blue.svg)
![License](https://img.shields.io/badge/license-MIT-green.svg)
![Status](https://img.shields.io/badge/status-beta-yellow.svg)

Reusable **FastAPI** engineering toolkit for Claude Code — Pydantic v2 schema design, async routing & dependency injection, SQLAlchemy/SQLModel data layer, Alembic migration safety (zero-downtime), pydantic-settings/12-factor config, security auditing, testing, and async-correctness/performance. Generic and **adapter-driven**: app layout, ORM, async-vs-sync stack, migration tool, and test runner are project-supplied inputs, not assumptions.

The plugin is opinionated about the things that bite FastAPI projects in production — a blocking call inside an `async def` that stalls the whole event loop, a table model returned straight as a response (leaking every column), N+1 lazy loads (and the async `MissingGreenlet` trap), unverified JWT signatures, wildcard CORS with credentials, autogenerated Alembic revisions applied unread, and committed secrets — and stays out of your way otherwise.

## Skills

Auto-activating from natural-language symptoms (no command needed). None are user-invocable; they fire when their domain comes up.

| Skill | Activates when… |
|-------|-----------------|
| `fastapi-pydantic` | Defining/editing a Pydantic model, validating request input, shaping a response, or when an endpoint leaks/over-accepts fields or mis-validates. Owns request-vs-response separation, validation layering, read/write field discipline, and mass-assignment prevention. |
| `fastapi-routing` | Writing/reviewing a path operation, an `APIRouter`, a `Depends()` dependency, or a response/status declaration — or when errors are inconsistent, a list is unbounded, or a status code is wrong. Owns route organization, dependency injection (incl. `yield` lifecycle), `response_model`, error shaping, and bounded pagination. |
| `fastapi-database` | Defining an ORM model/relationship, writing/reviewing a query, managing the session, choosing a loader, or diagnosing duplicated/slow queries. Owns model design, request-scoped sessions, the N+1 rule (`selectinload`/`joinedload`), transactions, and the **async-session** correctness traps. |
| `fastapi-migrations` | Running Alembic autogenerate/upgrade, reviewing a revision, or planning a schema change against live data. Owns the autogenerate review gate (and what it misses), reversibility (a real `downgrade`), schema-vs-data split, and **expand-contract** sequencing. |
| `fastapi-config` | Editing a settings module, adding config, wiring an env var/`.env`, or standing up an environment. Owns the typed `BaseSettings` object, env-driven config, secret handling, and per-environment correctness (debug, `/docs` exposure, CORS). |
| `fastapi-security-audit` | Reviewing for security, hardening before release, or responding to a suspected vuln/leaked secret. Renders the **pass/fail security verdict** — auth/JWT, authz (missing dependency, IDOR, mass-assignment), injection, CORS, secret/docs/debug exposure, uploads, dependency CVEs. |
| `fastapi-testing` | Writing/reviewing tests, choosing the test client, overriding a dependency, or diagnosing flaky/slow/leaky tests. Owns sync `TestClient` vs `httpx.AsyncClient`, `dependency_overrides`, transactional test DB, boundary mocking, and query-count regression coverage. |
| `fastapi-async-performance` | A route/task is slow or throughput collapses under load, adding caching/background work, or planning for scale. Owns the sync-vs-async `def` rule, **never blocking the event loop**, background tasks vs a queue, caching + invalidation, pool sizing, and pagination at scale. |

## Commands

| Command | Description |
|---------|-------------|
| `/fastapi-scaffold [resource] [--crud] [--async]` | Scaffold a resource surface (ORM model + `Create`/`Read`/`Update` schemas + `APIRouter` CRUD + Alembic revision + tests), applying every skill. Plans the diff and waits for approval. |
| `/fastapi-migrate [--make\|--apply\|--plan\|--check]` | Drive the Alembic safety gate — autogenerate, review the revision and its SQL (incl. what autogenerate missed), check reversibility, then upgrade. Produces an expand-contract plan for risky changes; never blind-applies. |
| `/fastapi-test [path] [--cov] [--failed] [--async]` | Run pytest with the detected client (sync/async) and test overrides, classify failures (regression vs fragile vs env), and surface coverage gaps. |
| `/fastapi-security [path] [--deps] [--fix]` | Run a code/config audit for auth, authz, injection, CORS, secrets, and CVEs; severity-rated findings with fixes. `--fix` applies only unambiguous low-risk remediations. Defensive only. |

## Agents

Read-only, return findings tables — they report, they don't mutate.

| Agent | Use for |
|-------|---------|
| `alembic-migration-analyzer` | Classify unapplied Alembic revisions by lock/rewrite risk and reversibility before applying to live data; flag autogenerate gaps; produce an expand-contract step plan. |
| `fastapi-security-auditor` | Deep defensive security audit of a codebase/app; severity-rated findings with `file:line` and fixes. |
| `async-query-optimizer` | Hunt N+1s (incl. the async-session trap), event-loop-blocking calls in async routes, over-fetch, write loops, Python-side aggregation, and offset-pagination at scale, with evidence per finding. |

## Safety Hooks

| Hook | Event | Behavior |
|------|-------|----------|
| Version & layout detection | SessionStart | Detects FastAPI/Pydantic/SQLAlchemy/Alembic version and sync-vs-async posture; injects a context line. Read-only. |
| Risky-edit guard | PreToolUse (Write/Edit) | **ADVISORY** (never blocks) — nudges on a blocking call (`time.sleep`/`requests.*`/`urlopen`) inside `async def`, a hardcoded secret literal in a config file, `allow_origins=['*']` with `allow_credentials=True`, and an empty Alembic `downgrade()`. Silent on sync routes, env-driven config, and scoped CORS. |
| Destructive-command guard | PreToolUse (Bash) | **BLOCKS** `alembic downgrade base`, `dropdb`, and `DROP DATABASE` unless the override token `ALLOW_FASTAPI_DESTRUCTIVE` is present in the same command. **ADVISORY** on `alembic stamp`. |

## Usage

The skills activate automatically as you work (review a query → `fastapi-database`; touch a Pydantic model → `fastapi-pydantic`; edit config → `fastapi-config`). Each command detects the project's layout on first run (caching non-secret adapter inputs to `.fastapi-kit.local.json`), so you can run them directly in any FastAPI repo. Use the commands for explicit workflows and the agents for read-only sweeps:

```
/fastapi-migrate --check
/fastapi-security --deps
@async-query-optimizer audit the items router
```

## Requirements

- A FastAPI project (a `fastapi` dependency, or an app module constructing `FastAPI(...)`). SQLAlchemy/SQLModel and Alembic are optional — data and migration features activate only when present.
- Python 3 available for the hooks (stdlib only; no third-party deps).
- For runnable checks (Alembic commands, the test suite, `pip-audit`), a working project environment.

## Limitations

- The plugin **does not run `alembic upgrade` or destructive commands for you blindly** — that's by design.
- Security review is **defensive**: it describes vulnerabilities and fixes, never working exploits.
- `.fastapi-kit.local.json` is a local cache — add it to `.gitignore`.
- Skill rules are framework-level; project-specific conventions (base model, error envelope, auth scheme) are adapter inputs you supply.

## Development & testing

The plugin ships a pytest suite under [`tests/`](./tests/) covering plugin structure (file existence, valid `plugin.json` / `hooks.json`, marketplace registration, skill/command/agent front-matter) and the behavior of all three hooks (block/allow/advisory paths, override token, fail-open on bad input):

```
pytest fastapi-plugin/tests/ -q
```

Structural validation (no pytest required):

```
python validate_plugin.py fastapi-plugin
```
